Página Acessível com ARIA

Exemplo de página acessível com ARIA

Seção 1

- Enter ou Espaço -

Se a implementação permitir que apenas um painel seja expandido e se outro painel for expandido, esse painel será recolhido.

- Tab -

Move o foco para o próximo elemento focável

- Shift + Tab -

Move o foco para o elemento focável anterior

- Down Arrow -

Se o foco estiver em um cabeçalho sanfonado, move o foco para o próximo cabeçalho sanfonado

- Up Arrow -

Se o foco estiver em um cabeçalho sanfonado, move o foco para o cabeçalho sanfonado anterior

- Home -

Quando o foco está em um cabeçalho sanfonado, move o foco para o primeiro cabeçalho sanfonado

- End -

Quando o foco está em um cabeçalho sanfonado, move o foco para o último cabeçalho sanfonado

Seção 2

- Quando o Espaço for pressionado e a caixa de seleção estiver em foco -

Quando marcado, o elemento checkbox tem o estado aria-checked definido como true

Quando não marcado, o estado aria-checked está definido como false.

Se um conjunto de caixas de seleção for apresentado como um grupo lógico com um rótulo visível, as caixas de seleção serão incluídas em um elemento -

- com grupo de funções que possui a propriedade aria-labelledby definida como o ID do elemento que contém o rótulo.

Ativa o botão

Um botão é um widget que permite aos usuários acionar uma ação ou evento.

Um alerta é um elemento que exibe uma mensagem breve e importante de forma a atrair a atenção do usuário sem interromper sua tarefa.

É importante observar que, no momento, os leitores de tela não informam aos usuários sobre alertas presentes na página antes da conclusão do carregamento da página.
